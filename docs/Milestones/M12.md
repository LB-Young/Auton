# M12 — Multi-Agent（多代理协作）

**目标**：多 Agent 协作。

**状态**：✅ 已完成

---

## 1. 概述

M12 让主 Agent（main）能够将任务委托给专业子 Agent（sub-agent），实现并行处理和专业化分工。

### 1.1 核心概念

```
主 Agent（main）
  ├── AgentTool — 调用子 agent 的工具
  │       ├── agent_create — 创建并等待子 agent 完成
  │       └── agent_list  — 列出已配置的 agent
  │
  ├── 子 Agent A（explore）— 并行探索代码
  ├── 子 Agent B（coder）— 并行写代码
  └── 子 Agent C（reviewer）— 并行审查代码
```

### 1.2 与 M9 Tasks 的关系

M9 Tasks 是**独立后台任务**，状态持久化，可由用户随时查看/停止。
M12 Agents 是**委托子 Agent**，主 Agent 等待结果，用于并行分担工作。

| 维度 | M9 Tasks | M12 Agents |
|------|-----------|------------|
| 调用方式 | `task_create` 工具 | `agent_create` 工具 |
| 执行 | 独立，不等待 | 等待结果 |
| 状态 | 持久化 JSON | 内存中，生命周期跟随主会话 |
| 用户可见 | `/tasks` 命令 | `/agents` 命令 |
| 用途 | 后台批处理 | 并行专家协作 |

## 2. Agent 定义

### 2.1 文件格式（`~/.auton/agents/*.md`）

```yaml
---
name: explore
description: "代码探索专家。当需要全面分析项目结构、理解复杂模块、搜索多种模式时使用。"
model: MiniMax-M2.7
tools:
  - read
  - glob
  - grep
permissionMode: auto
maxTurns: 10
skills:
  - codebase-navigation
omitClaudeMd: true
---

你是代码探索专家。你擅长全面分析项目结构、理解模块边界、快速定位关键代码。
你专注于观察和总结，不做修改决策。
```

### 2.2 AgentDefinition 类型

```python
@dataclass
class AgentDefinition:
    name: str                           # 唯一标识
    description: str                    # 何时使用（供主 Agent 决策）
    system_prompt: str                  # 系统提示词
    model: str | None                   # 模型（None = 继承主 Agent）
    tools: list[str]                   # 允许的工具（None = 全部）
    disallowed_tools: list[str]         # 禁止的工具
    permission_mode: str                # 权限模式
    max_turns: int | None              # 最大轮次限制
    skills: list[str]                   # 预加载的 skill
    mcp_servers: list[str]             # 需要的 MCP server 名称
    background: bool                   # 是否后台运行
    isolation: str | None              # worktree 隔离模式
    source: str                         # builtin / user / project / plugin
```

### 2.3 内置 Agent

| Agent | 用途 | 何时使用 |
|-------|------|----------|
| `explore` | 代码探索专家 | 分析项目结构、理解模块、搜索 |
| `coder` | 编程专家 | 写代码、改代码 |
| `reviewer` | 审查专家 | review PR、代码审查 |
| `planner` | 规划专家 | 任务分解、方案设计 |

## 3. AgentManager

`auton/agent/manager.py` — 管理所有 agent 定义和生命周期。

### 3.1 职责

- 从 `~/.auton/agents/` 和 `.auton/agents/` 加载 agent 定义
- 维护活跃的 sub-agent 列表
- 提供 agent 检索（按名称、按工具可用性）

### 3.2 方法

```python
class AgentManager:
    def load_agents(self) -> list[AgentDefinition]
    def get(self, name: str) -> AgentDefinition | None
    def list_by_tools(self, tools: list[str]) -> list[AgentDefinition]
    def create_run(self, name: str, prompt: str, parent_session_id: str) -> AgentRun | None
    def get_run(self, run_id: str) -> AgentRun | None
    def abort_run(self, run_id: str) -> bool
    def list_runs(self, status: str | None = None) -> list[AgentRun]
```

### 3.3 AgentRun 状态机

```
pending → running → completed
                    ↘ failed
                    ↘ aborted
```

## 4. `agent_create` 工具

主 Agent 通过此工具调用子 agent。

### 4.1 输入 Schema

```json
{
  "name": "agent_create",
  "description": "创建并运行一个子 Agent 来完成特定任务",
  "input_schema": {
    "type": "object",
    "properties": {
      "agent": {
        "type": "string",
        "description": "Agent 名称（builtin 或用户定义的）"
      },
      "prompt": {
        "type": "string",
        "description": "给子 Agent 的任务描述"
      },
      "wait": {
        "type": "boolean",
        "description": "是否等待完成（默认 true）",
        "default": true
      }
    },
    "required": ["agent", "prompt"]
  }
}
```

### 4.2 输出

```json
{
  "run_id": "a1b2c3d4",
  "agent": "explore",
  "status": "running",
  "message": "Agent 'explore' 已启动，正在分析项目结构..."
}
```

### 4.3 执行流程

```
AgentTool.execute(agent="explore", prompt="分析 auth 模块")
  → AgentManager.create_run("explore", prompt, session_id)
      → 创建 AgentRun，状态 = pending
  → asyncio.create_task(run_sub_agent(run_id))
      → 构建 system prompt（主 Agent 上下文 + 子 agent 专用 prompt）
      → 构建工具列表（按 agent 定义过滤）
      → SessionProcessor(system_prompt, tools, ...)
      → 流式返回结果
  → 返回 run_id（wait=true 则等待完成）
```

## 7. 子 Agent 执行上下文

子 agent 继承主 Agent 的：
- 项目根目录
- 工具列表（按 agent 定义过滤）
- Skill 列表
- MCP 客户端

子 agent 独立配置：
- System prompt（自定义 + 主 Agent 上下文摘要）
- 权限模式
- 最大轮次
