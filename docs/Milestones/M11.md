# M11 — Extensibility（可扩展性）

**目标**：可深度扩展的 Agent。

**状态**：✅ 已完成

---

## 1. 概述

M11 包含两部分能力：

1. **MCP 集成**（Model Context Protocol）— 通过 MCP 协议连接外部工具服务
2. **插件系统**（Plugin）— 热加载自定义工具包

---

## 2. MCP 集成（已完成基础实现）

### 2.1 架构

```
Auton Agent
    └── MCPTool（mcp 工具）
            └── MCPClient（per server）
                    ├── stdio JSON-RPC
                    └── MCP Server（npx 启动的子进程）
                            ├── github
                            ├── filesystem
                            └── ...（用户配置）
```

**已有实现**（`auton/tools/mcp/__init__.py`）：
- `MCPClient`：与 MCP server 通过 stdio JSON-RPC 通信
- `MCPTool`：统一透传工具，供 Agent 调用
- `load_mcp_servers()` / `stop_mcp_servers()`：生命周期管理

**本次补充**：
- `MCPConfig` 配置模型
- `MCPTool` 注册到工具注册表
- CLI 启动时初始化 MCP servers，退出时关闭
- `/mcp` 管理命令

### 2.2 配置格式

`~/.auton/config.yaml` 或 `.auton/config.yaml`：

```yaml
mcp:
  servers:
    - name: "github"
      command: ["npx", "-y", "@modelcontextprotocol/server-github"]
      env:
        GITHUB_TOKEN: "${GITHUB_TOKEN}"
    - name: "filesystem"
      command: ["npx", "-y", "@modelcontextprotocol/server-filesystem"]
      args: ["/Users/liubaoyang/allowed"]
```

### 2.3 工具调用

Agent 调用 MCP 工具时，通过 `MCPTool`：

```json
{
  "tool": "mcp",
  "server": "github",
  "tool": "create_issue",
  "arguments": {
    "owner": "myorg",
    "repo": "myrepo",
    "title": "Bug report"
  }
}
```

## 3. MCP 管理命令 `/mcp`

### 3.1 子命令

| 子命令 | 说明 |
|--------|------|
| `/mcp list` | 列出所有已配置的 MCP server 及其工具 |
| `/mcp status` | 显示每个 server 的运行状态（running/stopped/error） |
| `/mcp start <name>` | 启动指定 server（如未运行） |
| `/mcp stop <name>` | 停止指定 server |
| `/mcp add <name>` | 引导添加新 server（读取 YAML 配置片段） |
| `/mcp remove <name>` | 从配置中移除 server |

### 3.2 实现文件

- `auton/commands/mcp_cmd.py`

## 4. CLI 集成

### 4.1 启动流程

```
auton start
    → load_mcp_servers(config.mcp)
        → MCPClient.start() × N
        → registry.set_mcp_client(name, client)
    → MCPTool 已注册（registry）
```

### 4.2 退出流程

```
CLI exit
    → stop_mcp_servers()
```

## 8. 已知限制

- MCP server 通过 stdio 启动，不支持 HTTP/SSE 模式（未来 M11+ 支持）
- MCP server 进程崩溃后不会自动重启
- MCP 工具 schema 通过 `inputSchema` 字段透传，LLM 需支持动态 schema
